1. LOGING ON


After launching client aplication we need to choose if we will be working in online mode or in offline mode. For work in online or offline mode is necessary to log in by active login and password. When we are going to wok in online mode we need to specify server address and port. Next we must choose if we want to download data from server or send data to server (if there were changes in offline mode). For users without account there is an option to register after pressing register button.



2. REGISTRATION

In registration dialog window user must enter login (which is not used by another user), name, surname and password into appropriate text fields. After pressing confirm button, data are send to server which will confirm or deny registration of new user. When registration is successful, client is automatically logged in and his work with calendar can begin. If registration fails, an error message displays and user is back in log in and registration window. Registration is possible only in online mode.


3. CALENDAR


3.1 ENVIRONTMENT


After successful login we can see 3 cards in main window of which 2 represents weekly and monthly view on calendar (week, month) and third card represents agenda (user and group events).

In weekly and monthly view is possible to use buttons (previous week, next week) to view previous week or next week and button today to move calendar on actual date. Actual date is displayed in lower right corner of the aplication. Weekly view is divided to 7 days and 24 hours. By every hour we can see how many events are scheduled at concrete hour. In monthly view we see single weeks and days in them. In every day number of events that are scheduled is shown.

(Aplication was developed for windows platform so there can be little visual bugs at other platforms)


3.2 WORKING WITH EVENTS


To display event we need to double click at concrete hour in day in weekly view or concrete day in monthly view. After double clicking, new dialog window with the list of user events and group events goes up. By appropriate buttons it is possible to add or remove user and goup events (group events can add and remove only the group leader and only in online mode). User can send his event to another user by button copy event. After double clicking at event, new dialog window with notes and alerts to that event shows up. We can add, remove or edit notes by buttons add, remove and edit. Alerts can be only added or removed. By double clicking at note or alert we can see more information about it.


3.3 ADDING EVENT


In dialog window for adding event we need to fill the name of event, date, whether this event is repeatable, interval timer and category. When everything is correctly filled in, we can press confirm button. If it’s a group event we need to fill the group name too.



3.4 COPYING EVENT


In dialog window for copying event we need to fill the login of user whom we want to send event. We accomplish confirmation by pressing confirm button. We must be in online mode to copy event.


3.5 ADDING ALERT TO EVENT


In dialog window for adding alert we need to fill the date, whether event is repeatable and interval timer. We accomplish confirmation by pressing confirm button.


3.6 ADDING OR EDITING NOTE


In dialog window for adding or editing note we need to fill or edit body of the note. We accomplish confirmation by pressing confirm button.


3.7 CREATING GROUP


In menu in goups we can create group and add member to group. To create group we need to choose group name that is not in use. When we want to add member, we write in appropriate field user login. Adding members can only the group leader. For creating groups and adding members we need to be in online mode.
